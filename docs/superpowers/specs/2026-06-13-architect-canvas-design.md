# Architect Canvas — 設計規格

日期：2026-06-13
分支：product/architect-canvas
狀態：已定案，進入實作

## 鐵則（不可違反）

1. 在 product/architect-canvas 分支實作，絕不覆蓋現有 index.html 或 capability-card-playground.html。
2. 產品是全新檔案 architect-canvas.html，單一自包含、零外部依賴、可離線、雙擊即開（file://）。
3. 只做兩個分頁：系統架構圖、軟體定位。其餘五個分頁不帶入。
4. 去識別化：漢昕真實姓名、真實評語、PMS/EIP 內部資訊一個字都不進產品。
5. 不接公司資料源、不做帳號、不做線上授權伺服器。資料全在 localStorage + 可匯出/匯入 JSON。

## 產品定位

賣給軟體架構師。核心命題：架構師輸入「我有哪些系統、誰負責、彼此怎麼依賴、現況如何」，三十分鐘內長出一張資料驅動的權威歸屬架構圖，加一段可帶走的軟體定位論述。買斷、離線、不必訂閱 AI。

賣的不是 HTML 殼，是「輸入→生成」的引擎與方法論。

## 兩個分頁

### 分頁一：系統架構圖（產品心臟）

每個系統一張卡，欄位：
- 系統名（必填）
- 負責人（可空）
- 所屬層級：應用層 / 系統層 / 輸出層 / 基礎層
- 上下游依賴：可選多個其他系統
- 狀態：現行 / 轉型中 / 停擺 / 缺位

自動分層排版規則（確定性，保證整齊）：
1. 同層級系統等距排成一橫列。
2. 層與層間固定垂直間距。
3. 同列內依「被依賴次數」排序，被依賴多者置中，減少連線交叉。
4. 依賴線從下游接到上游權威來源；hover 任一節點高亮其上下游。
5. 狀態色：現行=實線、轉型中=橙虛框、停擺=紅虛框、缺位=紅虛線。

排版引擎必須寫成純函式：輸入系統陣列 → 輸出節點 x/y 座標 + 連線清單。無 UI 依賴，可移植 WASM、可被混淆抽離。

### 分頁二：軟體定位（引導式論述產生器）

架構師回答結構化問題（系統群核心價值、重複開發風險在哪、哪些能力該集中為權威來源），工具用模板組成：
- 一段專業、架構師口吻的軟體定位論述（可複製貼進提案）
- 一張「能力歸屬原則表」（共用能力 / 權威來源 / 應取用系統 / 現況落差），沿用原作「新系統立案前先查這張表」的殺手鐧。

論述產生引擎同樣寫成純函式，與 UI 分層，可移植 WASM。

## 雙態交付

- 範例公司（虛構，去識別化）：預設載入，讓買家秒懂能產出什麼。
- 我的公司（空白）：一鍵清空成自己的。
- 範例資料：提供 1-2 個範例 JSON（跨國軟體公司、資安服務商），可一鍵匯入。

## 匯出

- 匯出 / 匯入 JSON（備份、換機、版控自己的架構）。
- 匯出架構圖為 PNG / SVG（貼進提案）。

## 打包與即用（拆箱體驗）

Gumroad 交付一個 ZIP：
- architect-canvas.html（產品本體，雙擊即用）
- START-HERE（一頁式入門：雙擊哪個檔、先看範例、清空成我的公司、匯出備份）
- examples/（1-2 個範例 JSON）

全程零安裝、零帳號、零網路、零訂閱。

## 防盜三層（不犧牲離線與買斷）

- 第一層 可追溯指紋：交付時嵌入買家 email/姓名，頁尾與匯出圖顯示授權；HTML 內散布隱形買家 ID 編碼，可追溯外洩來源。
- 第二層 加殼：
  - A（必達）：前端混淆加殼——字串加密、控制流打平、反 DevTools、domain lock、嵌入隱形指紋。
  - B（護城河升級）：排版引擎 + 論述引擎用 Rust 編成 WASM，HTML 只呼叫。
- 第三層 真護城河：持續更新（新版型、新論述庫、新產業範例包）。正版附一年免費更新；盜版停在舊版死檔。

## 出貨閘門（不讓護城河綁架發布）

雙軌並行：
- 軌道一（必達）：純 JS 完整產品 + A 混淆加殼。兩天內一定完成，是最壞情況的出貨底線。
- 軌道二（並行）：WASM 核心。引擎從第一行就寫成純函式模組，可 1:1 移植。

WASM spike 驗證：能否把 .wasm inline 進單一 HTML、雙擊 file:// 離線正常跑、資料不外流。
- spike 過 → WASM 上殼為發布版。
- spike 不過或手感不如 JS 版 → 發布純 JS + A 混淆版，WASM 列為 v1.1 免費更新。

## 實作紀律（約束 agent）

- UI 與引擎嚴格分層。兩顆引擎（排版、論述）寫成無 UI 依賴的純函式模組。
- 授權/指紋檢查散布而非集中（集中易被一刀拔除）。
- 開發階段先寫未混淆、可讀、可測的版本；混淆是交付前的 pipeline，不混進原始碼。
- 沿用原作零依賴、inline、可離線的單檔哲學。
